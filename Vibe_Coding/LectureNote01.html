Agentic AI 효율 극대화를 위한 구조화된 기술 명세 프레임워크

발표자: 김동욱

SLIDE 1: Vibe Coding 이해: 추상적 '바이브'와 AI의 만남

  • 1. 바이브 코딩(Vibe Coding)의 개념

    • LLM 의존적 개발: 코드 생성을 위해 대규모 언어 모델(LLM)에 크게 의존하는 새로운 개발 패러다임입니다
    • 배경: 2025년 2월, OpenAI 공동 창업자 Andrej Karpathy에 의해 처음 언급되었습니다

  • 2. '바이브'의 전달과정과 난제

    • 대화형 정제: 개발자가 "모던한 느낌으로" 또는 "애니메이션 속도를 약간만 빠르게"와 같이 대화(Vibe)를 통해 코드를 점진적으로 완성
    • AI의 난제: LLM은 모호한 입력에 취약하며, 추상적인 '바이브'는 부정확한 코드 생성('환각')을 유발할 수 있습니다

SLIDE 2: PRD의 전략적 역할: AI 컨텍스트 엔진으로 포지셔닝

"PRD는 이제 인간 팀원 간의 문서가 아닌, AI의 코드 생성 루프를 통제하는 핵심 입력입니다."

  • 역할 전환: LLM의 코드 생성(Generation), 검토(Review), 리팩토링(Refinement) 순환 과정에 필요한 명확한 초기 컨텍스트를 제공하는 AI 컨텍스트 엔진 역할 수행.
  • 모호성 제거: 추상적인 '바이브'를 디자인 토큰, 아키텍처 원칙, 명명 규칙 등 AI가 오류 없이 처리 가능한 구체적인 기술 명세로 전환해야 합니다.
  • 구조적 통제: PRD가 전체 시스템의 매크로 설계(아키텍처, 기술 스택)를 정의하여, AI가 전체 구조를 임의로 변경하지 않도록 선행 가이드 역할을 합니다.

SLIDE 3: AI 소비 최적화를 위한 PRD 구조화 원칙

목표: LLM의 토큰 낭비 방지 및 핵심 정보 즉각 추출

  • 구조화된 형식(Structured Format): LLM이 쉽게 처리할 수 있도록 Markdown 사용을 강력히 권장합니다.
  • 명확한 섹션 구분(Clear Sections): 헤딩을 사용하여 정보를 체계적으로 정리해야 합니다.
  • 필수 포함 섹션:
    • 문제 정의 (Problem Statement): 제품이 해결하려는 핵심 문제 정의.
    • 솔루션/기능 개요 (Solution/Feature Overview): 제품이 제공할 주요 기능의 개략적인 목록.
    • 사용자 스토리 (User Stories): 사용자 관점에서 기능을 정의 (예: As a [User], I want [Feature], so that [Goal] 형식 준수)
    • 기술 요구사항 (Technical Requirements): 기능을 구현하는 데 필요한 기술적 세부 사항 및 명세.
    • 수용 기준 (Acceptance Criteria): 각 기능이 "완료"되었음을 판단하는 테스트 가능한 조건.
    • 제약 조건 (Constraints): 기술적, 비즈니스적 비협상 조건(Non-negotiables) 명시.

SLIDE 4: 핵심 작성 기법: 정밀성을 높이는 3대 요소

  • 1. 샤프한 사용자 스토리(Sharp User Stories)

    표준 형식 준수: "사용자로서 나는 [기능]을 원한다, 그래서 [목표]를 달성할 수 있다" (`As a [User], I want [Feature], so that [Goal]`).[3]

  • 2. 명시적 수용 기준(Acceptance Criteria)의 역할

    • 테스트 가능성: 각 기능에 대해 테스트 가능하고 측정 가능한 조건을 구체적인 bullet points 형태로 나열.
    • AI 자가 검증: LLM이 스스로 코드를 "완료"했는지 판단하는 기준을 제공하여 불완전한 코드 생성을 방지.

  • 3. 비기능 요구사항(NFR) 및 제약 조건 통합성능 목표(예: 응답 시간), 특정 라이브러리 사용 제한, 데이터 암호화 및 규정 준수 등 비협상 조건(Non-negotiables)을 명확히 명시.

SLIDE 5: 심화 전략 1: DDVC (설계 문서 기반 코딩) 적용

DDVC: 초기 설계 및 문서 검토 시간(1단계)의 중요성을 극대화하는 방법론.

문서 종류 주요 역할 핵심 내용
PRD 제품의 목표, 목적, 개략적인 기능 정의 (사전 문서) 문제 정의, 목표, 사용자 스토리, 수용 기준
Requirements 문서 LLM에게 직접적인 개발 요청을 위한 상세 기술 실행 계획서 (모듈별 작성 권장) 모듈 개요, 기능 요구사항, 관련 코드, 파일 구조, 규칙 (Rules)

SLIDE 6: 심화 전략 2: 대규모 프로젝트 관리를 위한 PRD-RFC 연동

문제: 코드베이스가 방대하면 LLM 컨텍스트 과부하 및 환각 발생.

  • 전략: PRD(거시적 틀) 내에서, 개별 기능/변경 사항 구현을 위한 미시적 계획을 RFC(Request for Comments) 문서로 분리.
  • RFC의 역할: 특정 작업에 대한 집중적인 지침(focused guidance)을 제공하여 AI가 전체 복잡성에 압도되는 것을 방지.
  • RFC 필수 포함 요소:
    • 기술 설계 (Technical design): 시스템의 전반적인 아키텍처 원칙(예: MVC, Hexagonal), 폴더 구조 컨벤션, 코딩 표준 등 AI가 준수해야 할 구조적 틀.
    • 필요한 엔드포인트/함수 (Required endpoints/functions): 시스템 간 통신에 필요한 정확한 API 엔드포인트 구조와 필요한 함수 명세.
    • 데이터 모델 및 DB 스키마 변경 사항: 사용할 데이터 모델의 정의, 그리고 데이터베이스 스키마의 추가/변경 사항을 명확히 명세하여 데이터 일관성 확보.
    • 보안 고려 사항 및 종속성: 데이터 암호화 표준, 인증 메커니즘, 특정 규정 준수 요구사항, 그리고 프로젝트가 의존하는 외부 라이브러리 및 종속성 명세.

SLIDE 7: AI 행동 제어: 기술 설계 선행 및 규칙 명세

  • 1. 필수 선행 기술 설계 포함

    • 구조 통일: 폴더 구조, DB 스키마, API 엔드포인트 구조를 명확히 설계하고 문서에 포함.
    • 비일관성 방지: AI가 임의로 데이터 스키마를 결정하게 만들지 않도록 강제.

  • 2. AI가 준수할 규칙 설정 (Cursor Rules)

    코딩 표준, 명명 컨벤션, 파일/폴더 구조 컨벤션, 사용할 아키텍처 원칙(예: MVC) 등을 명시하여 코드 품질 및 유지보수성 확보.

  • 3. 자동 테스트 코드 생성 요구사항 통합 (선택)

    Requirements 문서에 UnitTest 코드 생성을 필수 요구사항으로 통합하여, LLM이 코드 생성 후 pytest 기반의 자체 유효성 점검을 수행하도록 강제.

SLIDE 8: 통합적 개발 흐름: DDVC 3단계 MVP 개발 프레임워크

단계 주요 활동 문서 활용 및 개발자 역할
1단계: 설계 PRD/Requirements 초안 작성, 기술 설계 확정 (DB/API/폴더 구조) 개발자는 AI가 생성한 문서 초안을 꼼꼼히 검토 및 수정 (규칙 확정 집중)
2단계: 코드 생성 Requirements 기반 코드 생성, UnitTest 코드 자동 생성/실행 코드 품질 확인 및 테스트 결과 분석. 설계 문서와의 싱크 맞추기 필수
3단계: 통합 및 검증 프런트엔드 생성 및 전체 기능 통합 테스트 코드 수정 시 관련 설계 문서(PRD, RFC)도 즉시 함께 수정하여 AI 컨텍스트 최신화

SLIDE 9: 결론: PRD 품질 = 코드 품질

  • 1. 개발자의 역할 재정립: 설계자 및 검토자

    바이브 코딩은 코딩 시간을 줄이는 대신, 초기 설계와 문서 검토의 품질에 프로젝트 성공을 비례하게 만듭니다. 설계에 집중해야 합니다.

  • 2. 컨텍스트 통제 전략: 'One Feature at a Time'

    AI에게 전체 코드베이스를 한 번에 수정하도록 요청하는 대신, RFC 기반으로 '단 하나의 특정 작업'만 구현하도록 컨텍스트를 극도로 제한합니다.

  • 3. 최적화된 프롬프트 패턴 활용

    "Based on this RFC document, Please implement/modify the following specific feature in my codebase, Here are the relevant files."

    (필요한 컨텍스트와 파일만 포함하여 AI의 정확도를 극대화합니다.)

  • PRD는 AI 개발 생태계에서 가장 중요한 '입력 통제 장치'입니다.
